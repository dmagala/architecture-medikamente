# Классификация данных и инструменты защиты

| Данные | В чём необходимость защиты | Инструменты защиты |
|--------|---------------------------|-------------------|
| **Медицинские данные (PHI)** |
| Диагнозы, история болезни, результаты анализов | Требования 152-ФЗ и медицинского законодательства РФ. | **At Rest:** AES-256 шифрование в PostgreSQL (pgcrypto), шифрование на уровне файловой системы, HashiCorp Vault для управления ключами<br>**In Transit:** TLS 1.3 для всех соединений, VPN для межсерверной коммуникации, mTLS для микросервисов<br>**Контроль доступа:** ABAC через Apache Ranger, обязательное логирование всех обращений в Elasticsearch<br>**Тегирование:** Apache Atlas с тегами `confidentiality:critical`, `data_type:phi`, `medical_secret:true` |
| Лекарственные назначения, протоколы лечения | Высококонфиденциальные медицинские данные, часть врачебной тайны. Необходимость защиты от несанкционированного доступа и модификации. | **At Rest:** AES-256 шифрование, контроль целостности через цифровые подписи<br>**In Transit:** TLS 1.3, валидация данных через API Gateway<br>**Контроль доступа:** RBAC для медицинских специалистов, версионирование изменений<br>**Тегирование:** `confidentiality:high`, `data_type:phi` |
| **Персональные данные (PII)** |
| Паспортные данные, СНИЛС | Критически конфиденциальные идентификационные данные. Используются для мошенничества, кражи личности. Требования 152-ФЗ, высокие штрафы при утечке. | **At Rest:** AES-256 шифрование, обфускация в интерфейсах<br>**In Transit:** TLS 1.3, mTLS для внутренних сервисов<br>**Контроль доступа:** ABAC с обязательным одобрением доступа, полное логирование всех обращений<br>**Тегирование:** `confidentiality:critical`, `data_type:pii`, `category:identity`<br>**Дополнительно:** Право на удаление по запросу субъекта (152-ФЗ) |
| ФИО, дата рождения, адрес прописки | Высококонфиденциальные персональные данные. В сочетании могут идентифицировать человека. Требования 152-ФЗ. | **At Rest:** AES-256 шифрование, обфускация для отображения в списках<br>**In Transit:** TLS 1.3 для всех соединений<br>**Контроль доступа:** RBAC с ролями (ресепшен, медицинские специалисты)<br>**Тегирование:** `confidentiality:high`, `data_type:pii`<br>**Обезличивание:** Для аналитики - замена на хешированные идентификаторы |
| Телефон, электронная почта | Среднеконфиденциальные контактные данные. Риск спама, фишинга, несанкционированных звонков. | **At Rest:** AES-256 шифрование<br>**In Transit:** TLS 1.3<br>**Контроль доступа:** RBAC для ресепшена, кассиров, медицинских специалистов<br>**Тегирование:** `confidentiality:medium`, `data_type:pii`, `category:contact`<br>**Обфускация:** Частичное скрытие при отображении |
| Полис ОМС | Высококонфиденциальные данные, связанные с медицинским обслуживанием. | **At Rest:** AES-256 шифрование<br>**In Transit:** TLS 1.3<br>**Контроль доступа:** RBAC для медицинских специалистов и ресепшена<br>**Тегирование:** `confidentiality:high`, `data_type:phi`, `category:insurance` |
| **Финансовые данные** |
| Данные банковских карт | Критически конфиденциальные данные, подпадающие под PCI DSS. Используются для мошеннических транзакций. Высокие штрафы и репутационные риски. | **At Rest:** AES-256 шифрование, хранение только последних 4 цифр, полное удаление CVV после обработки<br>**In Transit:** TLS 1.3, соответствие PCI DSS требованиям<br>**Контроль доступа:** ABAC - строгий контроль, только платёжный шлюз имеет доступ<br>**Тегирование:** `confidentiality:critical`, `data_type:payment_card`, `pci_dss:true`<br>**Соответствие:** PCI DSS |
| Суммы оплат, договоры на оказание услуг | Высококонфиденциальные финансовые данные. Коммерческая тайна, необходимость защиты от утечки конкурентам. | **At Rest:** AES-256 шифрование<br>**In Transit:** TLS 1.3<br>**Контроль доступа:** RBAC для кассиров, бухгалтеров, менеджмента<br>**Тегирование:** `confidentiality:high`, `data_type:financial`, `category:contract` |
| Бухгалтерские данные, зарплатные данные сотрудников | Высококонфиденциальные данные. Коммерческая тайна, персональные данные сотрудников (152-ФЗ). | **At Rest:** AES-256 шифрование, интеграция с 1С через зашифрованные каналы<br>**In Transit:** TLS 1.3, VPN для соединений с 1С<br>**Контроль доступа:** RBAC только для HR и бухгалтерии<br>**Тегирование:** `confidentiality:high`, `data_type:employee_data`, `data_type:financial` |
| **Данные сотрудников** |
| ФИО сотрудников, должности | Среднеконфиденциальные кадровые данные. Персональные данные по 152-ФЗ. | **At Rest:** AES-256 шифрование<br>**In Transit:** TLS 1.3<br>**Контроль доступа:** RBAC для HR, бухгалтерии, менеджмента<br>**Тегирование:** `confidentiality:medium`, `data_type:employee_data` |
| **Операционные данные** |
| Логи доступа к данным, аудит-трейлы | Критически важные для безопасности. Необходимы для расследования инцидентов, соответствия требованиям регуляторов. Должны быть защищены от модификации. | **At Rest:** Immutable storage, шифрование AES-256, отдельное хранилище с ограниченным доступом<br>**In Transit:** TLS 1.3 для передачи логов<br>**Контроль доступа:** Только IT-отдел и служба безопасности, строгий ABAC<br>**Тегирование:** `confidentiality:high`, `data_type:operational`, `immutable:true`<br>**Целостность:** Цифровые подписи для предотвращения модификации |
| Метаданные файлов, версии медицинских карт | Важные для аудита и соответствия требованиям. Необходимость отслеживания изменений. | **At Rest:** Версионирование через Git-like системы, шифрование метаданных<br>**In Transit:** TLS 1.3<br>**Контроль доступа:** RBAC с логированием<br>**Тегирование:** `confidentiality:medium`, `data_type:operational` |
| **Аналитические данные** |
| Обезличенные статистические данные | Низкая конфиденциальность после обезличивания | **At Rest:** Шифрование AES-256, дополнительное обезличивание<br>**In Transit:** TLS 1.3<br>**Контроль доступа:** RBAC для аналитиков, бизнес-аналитиков<br>**Тегирование:** `confidentiality:low`, `data_type:analytical`, `anonymized:true` |

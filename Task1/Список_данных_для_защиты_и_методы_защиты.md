# Список данных для защиты и методы защиты

## 1. Персональные данные (PII)

### 1.1 ФИО (Фамилия, Имя, Отчество)
**Уровень конфиденциальности:** Высокий  
**Категория:** Персональные данные  
**Методы защиты:**
- **Шифрование:** AES-256 для данных
- **Обфускация:** Частичное скрытие для отображения в списках
- **Обезличивание:** Замена на хешированные идентификаторы для аналитики
- **Контроль доступа:** RBAC - только авторизованные сотрудники
- **Тегирование:** `confidentiality:high`, `data_type:pii`, `category:name`

### 1.2 Дата рождения
**Уровень конфиденциальности:** Высокий  
**Категория:** Персональные данные  
**Методы защиты:**
- **Шифрование:** AES-256 для данных в покое
- **Обфускация:** Отображение только года рождения
- **Обезличивание:** Замена на возрастные группы для аналитики
- **Контроль доступа:** RBAC - только медицинские специалисты и ресепшен
- **Тегирование:** `confidentiality:high`, `data_type:pii`, `category:birthdate`

### 1.3 Телефон
**Уровень конфиденциальности:** Средний  
**Категория:** Контактные данные  
**Методы защиты:**
- **Шифрование:** AES-256 для данных в покое
- **Обфускация:** Отображение последних 4 цифр
- **Обезличивание:** Удаление для аналитики
- **Контроль доступа:** RBAC - ресепшен, кассиры, медицинские специалисты
- **Тегирование:** `confidentiality:medium`, `data_type:pii`, `category:contact`

## 2. Медицинские данные (PHI)

### 2.1 Диагнозы
**Уровень конфиденциальности:** Критический  
**Категория:** Медицинские данные (врачебная тайна)  
**Методы защиты:**
- **Шифрование:** AES-256
- **Обфускация:** Не применяется
- **Обезличивание:** Замена на категории заболеваний
- **Контроль доступа:** ABAC - только медицинские специалисты, лечащий врач пациента
- **Тегирование:** `confidentiality:critical`, `data_type:phi`, `category:diagnosis`, `medical_secret:true`
- **Дополнительно:** 
  - Обязательное логирование всех обращений
  - Алерты при массовом доступе
  - Право на удаление по запросу пациента

## 3. Финансовые данные

### 3.1 Сумма оплаты услуг
**Уровень конфиденциальности:** Средний  
**Категория:** Финансовые данные  
**Методы защиты:**
- **Шифрование:** AES-256
- **Обфускация:** Не требуется
- **Обезличивание:** Агрегирование для отчётов
- **Контроль доступа:** RBAC - кассиры, бухгалтеры, менеджмент
- **Тегирование:** `confidentiality:medium`, `data_type:financial`, `category:payment_amount`

### 3.2 Данные банковских карт
**Уровень конфиденциальности:** Критический  
**Категория:** Платёжные данные (PCI DSS)  
**Методы защиты:**
- **Шифрование:** AES-256
- **Обфускация:** Хранение только последних 4 цифр
- **Обезличивание:** Полное удаление после обработки платежа
- **Контроль доступа:** ABAC - строгий контроль, только платёжный шлюз
- **Тегирование:** `confidentiality:critical`, `data_type:payment_card`, `category:pci_dss`, `pci_dss:true`
- **Дополнительно:**
  - Соответствие PCI DSS
  - Не хранить CVV

### 3.3 Договоры на оказание услуг
**Уровень конфиденциальности:** Высокий  
**Категория:** Юридические документы  
**Методы защиты:**
- **Шифрование:** AES-256

- **Обезличивание:** Удаление для аналитики
- **Контроль доступа:** RBAC - ресепшен, бухгалтерия, юридический отдел
- **Тегирование:** `confidentiality:high`, `data_type:contract`, `category:legal_document`

## 5. Приоритеты внедрения защиты

### Приоритет 1 (Критично - немедленно):
- Критичные медицинские данные (диагнозы, история болезни, анализы)
- Критичные PII (паспорт, СНИЛС)
- Платёжные данные (банковские карты)

### Приоритет 2 (Высокий - 1-3 месяца):
- Высококонфиденциальные PII (ФИО, адрес, дата рождения)
- Договоры на оказание услуг
- Зарплатные данные сотрудников

### Приоритет 3 (Средний - 3-6 месяцев):
- Среднеконфиденциальные PII (телефон, email)
- Финансовые данные (суммы оплат)
- Кадровые данные (ФИО сотрудников)

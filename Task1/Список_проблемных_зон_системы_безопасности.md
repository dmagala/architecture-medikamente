# Список проблемных зон системы безопасности

## 1. Хранение данных

### 1.1 Проблема: Неструктурированное хранение конфиденциальных данных
**Описание:**
- Данные хранятся в Excel-файлах, JPG, PDF на общем диске
- Нет централизованного хранилища
- Нет классификации данных по уровню конфиденциальности
- Невозможно отследить, где находятся все копии данных

**Риски:**
- Несанкционированный доступ к файлам
- Утечка данных через копирование файлов
- Потеря данных из-за отсутствия структуры
- Невозможность контроля доступа на уровне файлов

**Критичность:** КРИТИЧЕСКАЯ

### 1.2 Проблема: Отсутствие шифрования данных
**Описание:**
- Данные хранятся в открытом виде
- Нет шифрования на уровне файловой системы
- Нет шифрования при передаче данных между системами

**Риски:**
- При компрометации сервера злоумышленник получает доступ ко всем данным
- Перехват данных при передаче по сети
- Утечка данных при физическом доступе к серверу

**Критичность:** КРИТИЧЕСКАЯ

### 1.3 Проблема: Нет резервного копирования
**Описание:**
- Не указано наличие системы резервного копирования
- Нет плана восстановления после сбоев
- Нет тестирования восстановления данных

**Риски:**
- Потеря всех данных при сбое сервера
- Невозможность восстановления после атаки
- Нарушение непрерывности бизнеса

**Критичность:** ВЫСОКАЯ

## 2. Контроль доступа

### 2.1 Проблема: Слабая аутентификация и авторизация
**Описание:**
- Только доменная аутентификация (Active Directory)
- Нет многофакторной аутентификации
- Нет контроля доступа на уровне данных (RBAC/ABAC)
- Все пользователи домена имеют потенциальный доступ ко всем данным

**Риски:**
- Компрометация учётной записи
- Невозможность ограничить доступ на основе роли или атрибутов
- Внутренние угрозы

**Критичность:** КРИТИЧЕСКАЯ

### 2.2 Проблема: Отсутствие принципа наименьших привилегий
**Описание:**
- Нет разделения прав доступа между отделами
- Сотрудники ресепшена могут иметь доступ к финансовым данным
- Бухгалтеры могут иметь доступ к медицинским данным
- Медицинские специалисты могут иметь доступ к финансовым данным

**Риски:**
- Утечка данных через сотрудников, которым не нужен доступ
- Сложность аудита 

**Критичность:** ВЫСОКАЯ

### 2.3 Проблема: Нет сегментации сети и данных
**Описание:**
- Все данные в одной сети
- Нет изоляции критичных систем

**Риски:**
- Компрометация одной системы

**Критичность:** ВЫСОКАЯ

## 3. Аудит и мониторинг

### 3.1 Проблема: Отсутствие аудита доступа к данным
**Описание:**
- Нет логирования доступа к файлам
- Нет отслеживания изменений данных
- Нет мониторинга подозрительной активности
- Нет системы алертинга при нарушениях

**Риски:**
- Невозможно определить, кто получил доступ к данным
- Невозможно расследовать инциденты
- Невозможно обнаружить утечки данных
- Невозможно доказать соблюдение требований регуляторов

**Критичность:** КРИТИЧЕСКАЯ

## 4. Интеграции и передача данных

### 4.1 Проблема: Небезопасная передача данных
**Описание:**
- Нет защиты данных при передаче между системами
- Нет валидации данных на входе

**Риски:**
- Перехват данных при передаче
- Модификация данных злоумышленником
- Внедрение вредоносного кода через невалидируемые данные

**Критичность:** ВЫСОКАЯ

### 4.2 Проблема: Отсутствие API-безопасности
**Описание:**
- Нет аутентификации и авторизации API
- Нет валидации входных данных

**Риски:**
- Несанкционированный доступ через API
- Атаки на API (SQL injection, XSS)
- Утечка данных через неправильно настроенные API

**Критичность:** ВЫСОКАЯ

## 5. Обработка данных

### 5.1 Проблема: Ручная обработка данных
**Описание:**
- Все операции выполняются вручную
- Высокий риск человеческих ошибок
- Дублирование данных в Excel и 1С

**Риски:**
- Ошибки при вводе данных
- Утечка данных через ошибки сотрудников
- Неконсистентность данных между системами

**Критичность:** СРЕДНЯЯ

## 6. Соответствие требованиям

### 6.1 Проблема: Несоответствие требованиям 152-ФЗ
**Описание:**
- Нет политики обработки персональных данных
- Нет согласий на обработку данных
- Нет механизма удаления данных по запросу субъекта

**Риски:**
- Штрафы от Роскомнадзора
- Судебные иски

**Критичность:** КРИТИЧЕСКАЯ

### 6.2 Проблема: Несоответствие требованиям медицинского законодательства
**Описание:**
- Нарушение врачебной тайны

**Риски:**
- Потеря лицензии
- Репутационные потери

**Критичность:** КРИТИЧЕСКАЯ
